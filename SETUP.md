## 2. Подключение Supabase (чтобы изменения видели все посетители)

1. Зарегистрируйтесь на [supabase.com](https://supabase.com) и создайте новый проект.
2. Откройте **SQL Editor** в панели проекта и выполните целиком файл `supabase-schema.sql` из корня этого проекта — он создаст все нужные таблицы (`teams`, `players`, `matches`, `votes`, `faq`, `settings`), настроит политики безопасности (RLS), **включит Realtime** и заполнит начальными данными.
> **Важно для realtime (обновления в реальном времени):**
> Файл `supabase-schema.sql` автоматически включает realtime для всех таблиц. Если вы уже выполнили схему ранее и realtime не работал — выполните файл `supabase-realtime-migration.sql` в SQL Editor. Он добавит таблицы в publication `supabase_realtime`, после чего обновления от админ-панели будут мгновенно появляться у всех посетителей без перезагрузки страницы.

### 2.1. Вкладка NODBET (ставки, рулетка, NOD-Коины, Топ Хайроллеров)

Чтобы баланс NOD-Коинов, ставки, спины рулетки, никнеймы и Топ Хайроллеров были общими для всех пользователей и обновлялись в реальном времени — выполните в **SQL Editor** файл `supabase-nodbet-migration.sql`. Он создаёт таблицы `nodbet_profiles` (баланс, опыт, бусты, никнейм — **без почт**, чтобы топ их не раскрывал), `nodbet_bets`, `nodbet_roulette_spins`, настраивает RLS и включает для них realtime.

- Каждый игрок видит и меняет только свой баланс/ставки/спины, а топ хайроллеров читают все посетители.
- Никнейм меняется в профиле справа сверху (карандашик ✏️ рядом с именем). Разрешены русские и латинские буквы, цифры и символы «-», «_», «+»; пробелы запрещены. Никнеймы уникальны без учёта регистра.
- Если игрок никнейм не задал — в топе у него временно отображается случайный «фруктовый» ник вида `Апельсин_a1b2`.

> **Обновление баланса игры (честная рулетка, ставки, уровни, чистка накрутчиков).**
> Если база NODBET уже создана и работала со старой версией — выполните в **SQL Editor** ОДИН РАЗ файл `supabase-nodbet-fix-migration.sql`. Он:
> - убирает читерские бусты (VIP x3, страховки, x2 бустеры, «NODBET Pro»/золотой знак) у всех игроков;
> - разово наказывает накрутчиков: срезает баланс > 100 млн до ровно 100 млн и удаляет их ставки с суммой > 100 млн;
> - снимает искусственный лимит баланса (баланс становится `bigint` — растёт без потолка в ~2 млрд);
> - добавляет новые честные привилегии (дабл спин, рамка, свой статус, мультипас), прогноз овертайма в ставках и поля новой рулетки.

## 4. Логотипы команд

Логотипы лежат в `public/logos/team-a.png` … `team-f.png` (сгенерированы автоматически, в виде эмблем на тёмном фоне под стиль сайта). Заменить их можно двумя способами:

- Через админ-панель → вкладка «Команды» → загрузить файл (работает только при подключённом Supabase Storage, см. выше) или просто вставить прямую ссылку на изображение (URL) в поле «Логотип».
- Вручную — заменить файлы в папке `public/logos/` перед сборкой.

Рекомендуется использовать PNG с прозрачным или тёмным (#0D0D0D) фоном.

## 5. Публикация на GitHub Pages

1. `npm run build` — соберёт сайт в папку `dist/`.
2. Залейте содержимое `dist/` в ветку `gh-pages` (или настройте GitHub Actions деплой из `dist/`).
3. Роутинг сайта использует `HashRouter` (ссылки вида `#/matches`), поэтому GitHub Pages будет корректно открывать любые внутренние страницы без дополнительной настройки 404.html.

## 6. Как редактировать турнир изо дня в день

Всё редактируется через админ-панель (`/#/nb-admin-9991`):

- **Матчи** — создание матчей по этапам, ввод счёта, статуса (скоро/LIVE/завершён), ссылки на CYBERSHOKE. Очки в таблице считаются автоматически по формату (Bo1/Bo2/Bo3).
- **Команды** — название, состав (текстом), логотип, рейтинг силы.
- **Игроки** — K/D, Faceit Elo, рейтинг для таблицы лидеров, заметки.
- **FAQ** — вопросы/ответы внизу сайта.
- **Настройки** — заголовки главной страницы, объявление, текст приза, путь к PDF регламента.

Голоса зрителей (прогнозы/коэффициенты) на страницах матчей считаются автоматически из таблицы `votes` — вручную ничего считать не нужно, при желании голоса по конкретному матчу можно сбросить кнопкой в вкладке «Матчи».

## 7. Миграция «Server-Authoritative NODBET» (обязательно!)

Файл `supabase-server-authoritative-migration.sql` переводит всю экономику NODBET (баланс, рулетка, ставки, магазин, Двойная-Рулетка) на серверную сторону:

- **Зачем**: раньше исход спина и баланс вычислялись в браузере игрока и записывались в базу напрямую — любой мог подкрутить результат через DevTools. Теперь исходы генерирует Postgres (`random()` внутри SECURITY DEFINER функций), а клиенту запрещено менять денежные поля (RLS + GRANT/REVOKE).
- **Как применить**: Supabase → SQL Editor → выполнить файл ЦЕЛИКОМ **один раз, после всех остальных миграций**. Файл идемпотентен: повторный запуск безопасен.
- **Что меняется для игроков**: ничего визуально — те же шансы, выплаты, XP, цены магазина. Одиночная рулетка для гостей работает локально, как раньше; Двойная-Рулетка теперь требует вход в аккаунт (деньги двигает сервер).
- **Новые функции (RPC)**: `nodbet_spin`, `nodbet_claim_daily`, `nodbet_activate_promo`, `nodbet_buy_perk`, `nodbet_place_bet`, `nodbet_cancel_bet`, `nodbet_settle_my_bets`, `nodbet_reconcile_my_bets`, `nodbet_double_place_bet`, `nodbet_double_round_control`, `nodbet_double_spin`.
- **Важно**: код сайта (src/) этой версии рассчитан на эту миграцию — деплойте фронт и миграцию вместе. Без миграции онлайн-спины вернут ошибку, гостевой режим продолжит работать.

## 8. Миграция «Реакции в чате NODBET»

Файл `supabase-chat-reactions-migration.sql` добавляет под сообщениями чата лайки/дизлайки со счётчиками и эмодзи-реакции (с подсказкой «кто поставил» при наведении):

- **Требование**: сначала должна быть выполнена `supabase-chat-migration.sql` (таблица `nodbet_chat_messages`).
- **Как применить**: Supabase → SQL Editor → выполнить файл ЦЕЛИКОМ. Идемпотентен: повторный запуск безопасен.
- **Что создаёт**: таблицу `nodbet_chat_reactions` (кто, на какое сообщение и какую реакцию поставил) и серверную функцию `nodbet_chat_toggle_reaction` — повторное нажатие снимает реакцию; лайк и дизлайк взаимоисключающие; не больше 5 разных эмодзи от одного пользователя на сообщение.
- **Права**: смотреть реакции могут все посетители; ставить и снимать — только авторизованные и только через RPC (прямые INSERT/UPDATE/DELETE запрещены). Realtime включён — реакции появляются у всех мгновенно.
- **Без этой миграции** чат продолжит работать как раньше, но кнопки реакций будут показывать ошибку «Не удалось поставить реакцию».
